# Style guide inspiration

These are the style guides I’m using as reference points while building this
manual.

## Splunk Style Guide

- URL:
  [https://help.splunk.com/en/splunk-style-guide/welcome-to-the-splunk-style-guide](https://help.splunk.com/en/splunk-style-guide/welcome-to-the-splunk-style-guide)
- Why it’s useful:
  - Clear, practical guidance written for a documentation team
  - Strong on consistency, terminology, and how to write for product docs

## Microsoft Writing Style Guide

- URL:
  [https://learn.microsoft.com/en-us/style-guide/welcome/](https://learn.microsoft.com/en-us/style-guide/welcome/)
- Why it’s useful:
  - Opinionated standards at scale (voice, tone, grammar, accessibility)
  - Great reference for UI terms and consistent phrasing patterns

## Mintlify Guides

- URL:
  [https://www.mintlify.com/guides/introduction](https://www.mintlify.com/guides/introduction)
- Why it’s useful:
  - Modern docs UX patterns and documentation structure
  - Helpful examples for writing developer-friendly guides
