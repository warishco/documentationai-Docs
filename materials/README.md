# Resources

This folder is a private workspace for building the Writing Manual.

Nothing in this folder is meant to be deployed.

## What goes here

- Source links, snippets, and rough notes
- Extracted ideas and frameworks I want to test
- Draft checklists before they graduate into the manual

## Rules

- Keep files in `materials/` as `.md` (not `.mdx`).
- Do not add anything in `materials/` to `documentation.json`.

## Process

1. Start with the high-level plan in
   [working-tree.md](../archives/working-tree.md).
2. Once having a clear picture of the structure, move to
   [information-architecture.md](../archives/information-architecture.md) to
   create the content structure.
3. Create per-topic outlines in `materials/outlines/`.
4. Write the content for each page in the repository root or in a folder.
5. Add the pages to `documentation.json`.
6. Document the manual creation process in
   [writing-manual-devlog.md](writing-manual-devlog.md).
